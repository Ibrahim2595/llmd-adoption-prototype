---
title: "Disaggregated Serving"
description: "How llm-d separates prefill and decode phases across different hardware."
sidebar_order: 2
---

Disaggregated serving splits the LLM inference process into two independent phases — prefill and decode — that run on separate pods. Prefill workers process the input prompt and produce the initial KV cache, while decode workers stream token generation. This separation reduces time-to-first-token and enables more efficient hardware utilization for large models and long-context workloads.
